## 一、 前言与约定

本说明书详细解释"解救事件ini片段"系列文件的功能与使用方法。该系列包含两个文件：

- **解救事件ini片段.ini**：包含detect1-detect10的触发配置
- **解救事件ini片段11至20.ini**：包含detect11-detect20的触发配置

在阅读本文档前，请先了解以下约定：

- **detectX** 表示第X个检测触发组，例如 `detect1` 表示第一组检测，`detect2` 表示第二组，以此类推
- **RTB0000** 是所有detect共享的基础事件触发器，仅在第一个文件中定义
- **RTn0001** 表示detectn的主检测触发，**RTn0002-RTn0004** 表示对应的三个动作触发

## 二、 核心组件详解

### 1. 触发ID命名规范

所有触发ID遵循严格的命名规则：

| 触发类型  | 命名格式 | 示例    | 说明                     |
| --------- | -------- | ------- | ------------------------ |
| 基础事件  | RTB0000  | RTB0000 | 所有detect共享的基础事件 |
| 检测触发  | RTn0001  | RT10001 | detectn的主检测触发      |
| 动作触发1 | RTn0002  | RT10002 | detectn的第一个动作触发  |
| 动作触发2 | RTn0003  | RT10003 | detectn的第二个动作触发  |
| 动作触发3 | RTn0004  | RT10004 | detectn的第三个动作触发  |

### 2. 基础事件（RTB0000）

RTB0000是所有detect共享的基础事件，定义在**解救事件ini片段.ini**中：

| 节         | 配置内容                                                                                    | 说明                                                                  |
| ---------- | ------------------------------------------------------------------------------------------- | --------------------------------------------------------------------- |
| [Actions]  | `RTB0000=3,54,2,RTB0000,0,0,0,0,A,11,4,RescueText,0,0,0,0,A,19,7,CrateFreeUnit,0,0,0,0,A` | 包含三个动作：禁止自身触发、显示RescueText文本、播放CrateFreeUnit音效 |
| [Triggers] | `RTB0000=Americans,<none>,[RESCUE]Basic Event,1,1,1,1,0`                                  | 允许重复触发                                                          |
| [Events]   | `RTB0000=1,8,0,0`                                                                         | 触发条件为"任何事件"（无条件触发）                                    |
| [Tags]     | `RTB0000_tag=2,[RESCUE]Basic Event 1,RTB0000`                                             | 类型为2，关联到RTB0000触发                                            |

### 3. 检测触发组（detectX）

每个detectX包含一个检测触发和三个动作触发，以detect1为例：

#### 检测触发（RT10001）

| 节         | 配置内容                                                                                                  | 说明                                       |
| ---------- | --------------------------------------------------------------------------------------------------------- | ------------------------------------------ |
| [Actions]  | `RT10001=4,53,2,RTB0000,0,0,0,0,A,53,2,RT10002,0,0,0,0,A,53,2,RT10003,0,0,0,0,A,53,2,RT10004,0,0,0,0,A` | 允许RTB0000、RT10002、RT10003、RT10004触发 |
| [Triggers] | `RT10001=Americans,<none>,[RESCUE]Detect No.1,0,1,1,1,0`                                                | 名称为"Detect No.1"，允许重复触发          |
| [Events]   | `RT10001=1,48,0,0`                                                                                      | 触发条件为"被任何事物摧毁"（48号事件）     |
| [Tags]     | `RT10001_tag=0,[RESCUE]Detect No.1 1,RT10001`                                                           | 关联到RT10001触发                          |

#### 动作触发（RT10002-RT10004）

| 触发ID  | 配置内容                       | 说明                                     |
| ------- | ------------------------------ | ---------------------------------------- |
| RT10002 | `RT10002=1,14,0,0,0,0,0,0,A` | 14号结果"更改所属方"，将关联对象变更所属 |
| RT10003 | `RT10003=1,14,0,0,0,0,0,0,A` | 同上                                     |
| RT10004 | `RT10004=1,14,0,0,0,0,0,0,A` | 同上                                     |

## 三、 使用方法

### 1. 文件组织

- **解救事件ini片段.ini**：包含detect1-10和RTB0000基础事件
- **解救事件ini片段11至20.ini**：包含detect11-20（不含RTB0000，依赖第一个文件）

### 2. 基础事件调用

所有detect的检测触发（RTn0001）均通过53号动作"允许触发"调用RTB0000：

```ini
RTn0001=4,53,2,RTB0000,0,0,0,0,A,...
```

### 3. 自定义修改指南

#### 修改检测条件

要修改detectn的检测条件，需修改对应RTn0001的[Events]配置：

```ini
; 将"被摧毁"条件改为"进入区域"条件（24号事件）
RTn0001=1,24,0,0
```

#### 修改动作结果

要修改动作触发的行为，需修改RTn0002-RTn0004的[Actions]配置：

```ini
; 将"更改所属方"改为"播放语音"（21号结果）
RTn0002=1,21,6,EVA_ObjectiveComplete,0,0,0,0,A
```

#### 增加动作数量

如需增加动作数量，可在检测触发中添加更多53号动作：

```ini
; 增加允许RTn0005触发
RTn0001=4,53,2,RTB0000,0,0,0,0,A,53,2,RTn0002,0,0,0,0,A,53,2,RTn0003,0,0,0,0,A,53,2,RTn0004,0,0,0,0,A,53,2,RTn0005,0,0,0,0,A
```

## 四、 常见问题

### 1. 为什么detect11-20单独给出？

一般情况下10个已经够用，且导入后会打乱编号，按需导入"11-20"的ini

### 2. 如何添加detect21以上的触发？

按照命名规则扩展即可：

- detect21：RT210001-RT210004
- detect22：RT220001-RT220004
- 以此类推

### 3. 如何修改基础事件的文本和音效？

修改RTB0000的[Actions]配置
