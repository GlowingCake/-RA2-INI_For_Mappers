# 红警2 INI 提取格式化工具

## 功能介绍

本工具用于处理红警2/尤复的地图文件（.yrm/.ini），实现以下功能：

1. **提取**：从地图文件中提取触发器相关片段（Actions、Events、Triggers、Tags、TaskForces、ScriptTypes、TeamTypes、VariableNames 等）
2. **编号格式化**：将旧的八位十六进制 ID（如 `01000068`）转换为新的字母数字格式（如 `DEF0001`）
3. **不提取**："Header", "Preview", "PreviewPack", "Basic", "FA2spVersionControl", "Houses", "IsoMapPack5", "Lighting", "Map", "OverlayDataPack", "OverlayPack", "Digest"

***

## 文件说明

| 文件            | 说明                  |
| ------------- | ------------------- |
| `提取片段.py`     | 提取脚本，从地图文件中提取需要的片段  |
| `重命名.py`      | 重命名脚本，对提取的片段进行编号格式化 |
| `default.ini` | 默认配置文件，用于判断哪些是默认数据  |

***

## 使用方法

### 第一步：提取片段

```bash
python 提取片段.py "地图文件路径"
```

**示例：**

```bash
python 提取片段.py "c:\map\test.yrm"
```

**输出文件：** `{原文件名}-提取.ini`

***

### 第二步：重命名编号

```bash
python 重命名.py "提取后的ini文件"
```

**示例：**

```bash
python 重命名.py "c:\map\test-提取.ini"
```

运行后会依次询问三个配置项（可直接回车使用默认值）：

| 询问            | 说明                  | 默认值 |
| ------------- | ------------------- | --- |
| 请输入统一前缀（3位字母） | 用于触发器、小队等的命名        | DEF |
| 局部变量从几开始      | VariableNames 的起始编号 | 50  |
| 特遣和脚本从多少开始    | TaskForces 等的起始编号   | 500 |

**输出文件：** `{原文件名}-提取-重命名.ini`

***

## 编号规则

### 触发器系统（Actions / Events / Triggers）

- **格式**：`{前缀}{序号}`
- **序号**：从 0001 开始递增
- **示例**：`DEF0001`, `DEF0002`, `DEF0003`

### Tags（标签）

- **格式**：`{对应的Trigger新ID}_tag`
- **示例**：`DEF0001_tag`, `DEF0002_tag`

### 局部变量（VariableNames）

- **格式**：直接数字
- **起始**：用户指定（默认 50）
- **示例**：`50=Mission01,0`, `51=Mission02,0`

### 特遣小队 / 脚本 / 队伍类型

- **键名格式**：直接数字，从用户指定值开始（默认 500）
- **值格式**：`{前缀}{原ID后4位hex}`
- **示例**：`500=DEF0065`, `501=DEF0082`

### 内容 Section

- **以 01 开头的旧 Section 名**：重命名为 `{前缀}{后4位hex}`
- **已有命名（如 MT00001、RST1001）**：保持不变
- **值中的旧 ID 引用**：全部自动更新为新 ID

## 已有命名保护

如果文件中已经有格式化好的命名（如 `MT00001`、`OB10001`、`RST1001` 等），这些不会被重命名，只会更新其中引用的旧 ID。

**示例：**

- `[MT00001]` 保持不变，但其中的 `01000068` 会更新为新的 ID
- `[RST1001]` 保持不变，但其中的 `Script=01000090` 会更新为 `Script=DEF0090`

***

## 注意事项

1. **编码问题**：脚本支持 UTF-8、GBK、GB2312 编码自动识别
2. **空值行**：文件中形如 `=` 的空值行会被自动跳过
3. **备份建议**：处理前建议备份原文件
4. **引用更新**：所有跨 Section 的引用都会被自动更新

***

## 快速开始

1. 将地图 .yrm 文件拖到 `提取片段.py` 上
2. 运行 `重命名.py`，按提示输入前缀（直接回车使用默认 DEF）
3. 在同目录下获取 `-提取-重命名.ini` 文件
