# 红警2 INI提取与编号格式化

## 脚本位置

- 提取脚本：`scripts/提取片段.py`
- 重命名脚本：`scripts/重命名.py`
- 默认配置：`default.ini`（与scripts同目录）

## 使用方法

```bash
# 步骤1：提取片段
python scripts/提取片段.py <输入文件>

# 步骤2：重命名
python scripts/重命名.py <提取后的ini文件>
```

## 用户询问配置（必须执行）

### 第一次询问：统一前缀

- **问题**：`请输入统一前缀（3位字母，默认DEF）:`
- **说明**：触发器、特遣小队、脚本等统一使用此前缀
- **默认**：DEF
- **验证**：必须为3位字母，否则使用默认值

### 第二次询问：局部变量起始

- **问题**：`局部变量从几开始（默认50）:`
- **说明**：VariableNames的起始编号
- **默认**：50

### 第三次询问：特遣和脚本起始

- **问题**：`特遣和脚本从多少开始（默认500）:`
- **说明**：TaskForces、ScriptTypes、TeamTypes的键名起始编号
- **默认**：500

## 统一前缀命名格式

| 类型 | 格式 | 示例 |
|------|------|------|
| 触发器（Actions/Events/Triggers） | {前缀}{序号} | DEF0001, DEF0002 |
| Tags | {对应TriggerID}_tag | DEF0001_tag |
| 内容Section（以01开头） | {前缀}{原ID后4位hex} | DEF0082, DEF0090 |

**序号规则**：
- 触发器相关：按出现顺序从0001开始递增
- 特遣/脚本/队伍：键名从用户指定数字开始递增

## ID关联关系与编号规则

| Section | 键名格式 | 值格式 | 起始值 |
|---------|---------|--------|--------|
| [Actions] | {前缀}{序号} 或保持已有 | - | 0001 |
| [Events] | {前缀}{序号} 或保持已有 | - | 0001 |
| [Triggers] | {前缀}{序号} 或保持已有 | - | 0001 |
| [Tags] | {对应Trigger}_tag | - | - |
| [VariableNames] | 数字 | - | 用户指定 |
| [TaskForces] | 数字 | {前缀}{原ID后4位hex} | 用户指定 |
| [ScriptTypes] | 数字 | {前缀}{原ID后4位hex} | 用户指定 |
| [TeamTypes] | 数字 | {前缀}{原ID后4位hex} | 用户指定 |
| [内容Section] | 以01开头的重命名，其他保持 | 值中旧ID引用需转换 | - |

## 算法核心原则

**先建立完整映射表，后统一替换**

1. **第一遍扫描**：建立旧ID→新ID映射表（只处理以01开头的旧ID）
2. **第二遍扫描**：根据映射表替换所有值中的旧ID引用
3. **已有命名保持不变**：如 MT00001, OB10001, RST1001 等不重命名

## 输出文件

1. `{原文件名}-提取.ini` - 提取后的原始ID版本
2. `{原文件名}-提取-重命名.ini` - 格式化后的新ID版本
